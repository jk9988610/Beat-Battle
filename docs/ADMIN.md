# 管理员说明

## 谁能当管理员

两种方式（在 `js/config.js` 中配置）：

| 方式 | 配置项 | 用法 |
|------|--------|------|
| **固定昵称** | `ADMIN_USER_NAMES` | 加入赛季时昵称填 `管理员` 或 `主持人`（可自定义列表） |
| **口令验证** | `ADMIN_PIN` | 任意昵称登录后，在主页「管理员验证」输入口令 |

口令仅对**当前浏览器标签页**有效，关闭页面后需重新验证。固定昵称在各设备登录均为管理员。

## 管理员能做什么

- 推进赛季阶段（报名 / 上传 / 评阅 / 揭晓）
- 结束本赛季并开启下一季
- 导出 / 导入赛季备份

普通参与者只能：上传、评阅、查看排名（揭晓后）。

## 如何指定新的管理员

编辑仓库 `js/config.js`：

```javascript
export const ADMIN_USER_NAMES = ['管理员', '主持人', '你的名字'];
export const ADMIN_PIN = '改成只有主持人知道的口令';
```

保存并推送到 `main`，等 GitHub Pages 更新后生效。

## 安全说明

当前为前端校验，适合班级/活动场景。`ADMIN_PIN` 会出现在公开代码中，请使用**活动专用口令**并勿存放敏感信息。若需更强安全，需接入 Supabase Auth 与服务端策略。
